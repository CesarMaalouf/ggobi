This file gives a list of the changes for each of the "versions" of GGobi. It is organized in reverse order so that the most recent releases are We hope to keep it as up-to-date as possible, but we often forget.

1.1-2

Configuration file for use with pkg-config for ggobi
pkg-config --cflags ggobi, etc. works include pkg-config --variable=Home ggobi
Added the missing CMD for R CMD INSTALL Rggobi.
Install Rggobi if the Rggobi directory is present.
To override this, use ./configure --with-R=no

1.0.0

  • Color schemes now read by default, from the file data/colorschemes.xml
    We use an explicit ggobiHome in the Session options to identify the location of the GGobi "installation".
    On Windows, GGobi is added to the registry in HKEY_LOCAL_MACHINE\SOFTWARE\GGobi and has the key Home giving the location in which it is installed.
    This is used to find GGobi files (e.g. ggobirc, plugins), especially when GGobi is embedded in other applications and the location cannot be computed from the name of the running application.

    • ExcelReader plugin for reading data directly from Excel.

    Configuration now tries -shared and if that fails -G as the flag for creating a shared library..
    This should help things on Solaris with a GCC that wants -G.

    --plugin=fileName allows one to specify individual plugin.xml files directly on the command line
    This is useful for debugging, etc. without having to edit the ggobirc file to add or remove plugin entries.
    Major internal changes of the data structures making them use the Gtk class system and now using reflectance.
    restore argument allows one to restore a collection of plots for a given ggobi from a previous session.
    --restore=file and -restore file
    Selecting a display in the Display Tree now raises the window to the foreground.
    Events are defined on ggobid, datad structures rather than the associated widgets/windows.
    Plugins may need to be updated.

    Version 0.99-1

    Support for new XML data format in which values are specified by a <real> or <int> element within a <record>. Missing values are represented by the element <na/>.
    The old format in which the values are given as a simple string within the version is also supported.
    Specifying the active color scheme within a data file.
    Support for specifying the colors cheme within a data file to apply to the given GGobi instance using <activecolorscheme>. One can use a name attribute to identify the scheme by name from with the available schemes. Additionally one can use file to specify the name of an XML file containing color schemes to be read and added to those available to the GGobi instance. This file is read before the name is used to resolve the colorscheme.
    In the future, we will allow the color scheme to be in-lined within the file.
    The Color Schemes tool uses a tree to display the different color schemes, grouped by type.

    Version 0.99

    Perl language plugin allowing one to write general and input plugins in Perl.
    Redesigned the way missings are viewed: instead of creating special 'missings' displays, we create a new dataset and link it to the original dataset.
    Implemented rescaling in response to dragging in the scatterplot axes.
    1D plots: implemented cycling, added an option to add line segments to ASHes. (Added that same option to the 1D tour, too.)
    Variable selection panel: switched from a single checkbutton to two togglebuttons and a button. We hope this is a simpler interface for new users to figure out.

    Version 0.98

    Integrated the use of so-called colorschemes throughout the code; the initial scheme can be defined on the command line like this: --activeColorScheme="schemename"
    Support for bidirectional edges: if an edge is drawn from a to b and from b to a, then draw half of the edge at a time. In that way, each edge can be a different width or color.

    Version 0.92.0

    A variable within a dataset can be used as the labels for the records. <realvariable> and <categoricalvariable> support recordLabel.

    Version 0.9.6

  • Additions to the preferences specifiable in an initialization file.
    <autoplot>, <allowNoDisplays>, <quitOnLastGGobi> Each has a on attribute that can be a logical value (TRUE, true, True, T, 1 or FALSE, False, false, F, 0)
    Also, <numDefaultPlotVars> sets the default number of variables to display in the different plots.
    <sessionFile> gives the name to which the session is saved.
  • <autoplot on="true|false">
    controls whether the initial scatterplot is created and displayed for a newly created GGobi instance with data.
  • <allowNoDisplays>
    controls whether one can close the last display for a GGobi instance and therefore have no plots displayed, but only the main GGobi window for that instance.
  • <quitOnLastGGobi>
    controls whether closing the last remaining GGobi instance main window terminates the GGobi session.
  • <numDefaultPlotVars>
    The attributes are scatmat, parcoords. The value for each gives the default number of variables to display in the display. If this is negative, all variables are used.
  • <sessionFile>
    The attributes are name which gives the name of the file to which to write the session information, and compress which is a non-negative integer value specifying the compression level. Since the file is short, 0 is a suitable value to allow the file to be read as regular text.
  • Added an item `Close' to the File menu that closes the particular GGobi instance.
    Quit terminates the application.
  • Java meta-plugin that allows developers to create plugins implemented in Java.
    Examples in the plugins/JVM directory.
  • Extended facilities in the DataViewer plugin to link point brushing, identification and movement to the cells in the worksheets. Also changes made in a cell are updated in the GGobi dataset and plots.
  • GGobi plugin for Gnumeric allowing ggobi to be used in spreadsheets.
  • Database connectivity implemented via plugins for Postgres and MySQL.
  • Meta plugin using R that allows one to write plugins for GGobi using the S language.
  • Splash screen has version information.
  • Updates for the Tree of displays.
  • Updates dynamically to reflect changes in the variables displayed in a plot.
  • Displays the name of the dataset from associated with the variables in the plot.
  • Selecting the node for that display (i.e. one of the top-most nodes) causes the first plot in that display to become active.
  • Storing the session queries the user for the name of the file to which the session information should saved.
  • The session information now includes all ggobis, the preferences, input sources (i.e. files, URIs, etc.) visited, etc.
  • Thanks to John Plummer for detailed bug reports.
