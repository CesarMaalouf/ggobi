When working with the CVS repository (either directly or using rsync), there are two ways to compile the code.
  • As a ggobi distribution.
    • From within the GGobi directory, create the usual distribution format via the command 
       make ADMIN=1 NO_XGOBI_GROUP= DOC_TEX_FILES=
      This creates and populates the relevant directories in /tmp/ggobi/Install/ggobi.
    • cd /tmp/ggobi/Install/ggobi
    • Type make (or gmake if GNU make is not the default version of the make utilit on your machine).
    • All should have built.
  • In the check-out directory.
    • Change to the ggobi directory.
    • Type make local.config
    • Edit the local.config file to provide any necessary information, etc. for your system.
    • Type make (or gmake) to build the stand-alone ggobi executable.
    • To create GGobi as a shared library, type 
      	      make libggobi.so
