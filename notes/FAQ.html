    Installation

  • Why do I see G_INLINE_FUNC in the compilation?
    This is a mechanism to help the header files in the glib library to compile inlined functions. Do a web search on G_INLINE_FUNC to find out more information. See the next question for symptoms of what can go wrong if this is not defined when it is needed.
  • I get error messages when compiling the C code about the glib.h header file that look something like 
    "/home/duncan-jessie/usr/include/glib-1.2/glib.h", line 1573: undefined or not a type: inline
"/home/duncan-jessie/usr/include/glib-1.2/glib.h", line 1574: parameter not in identifier list: g_bit_nth_lsf
"/home/duncan-jessie/usr/include/glib-1.2/glib.h", line 1576: only "register" valid as formal parameter storage class
"/home/duncan-jessie/usr/include/glib-1.2/glib.h", line 1576: parameter not in identifier list: inline
    This means that your compiler cannot inline code and so this must be turned off. Do this by adding 
           -DG_INLINE_FUNC=extern
    to the definition of the make variable OTHER_CFLAGS in the Makeconf file in the top-level directory. Please also send mail about this to ggobi-help@ggobi.org and tell us about your OS and compiler so we can add this to the configuration script.
